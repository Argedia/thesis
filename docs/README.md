# Documentación

## Estructura

- `proyecto/`: documentación técnica del sistema, arquitectura, contexto y notas internas de desarrollo.
- `tesis/`: documentación académica y anexos usados para el documento de tesis.

## Criterio de separación

- Si el documento describe implementación, arquitectura, refactorización o contexto técnico del software, va en `proyecto/`.
- Si el documento sirve como insumo directo para el entregable, anexos, validaciones o argumentación académica, va en `tesis/`.
